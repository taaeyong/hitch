# Demo Flow

## Core Demo Story
The intended v4 demo should make the system feel like a living relationship operating system rather than a one-off chatbot.

## Suggested Flow
1. Telegram session starts
2. User enters onboarding
3. Relationship is selected
4. Partner invite is shown as optional and skippable
5. Relationship setup is completed
6. First question or scenario begins
7. Hitch produces interpretation + simulated partner perspective
8. Wiki grows visibly
9. Weekly/report/graph-facing artifacts update
10. Demo-safe visible artifacts may be inspected

## Important Notes
- the Telegram session start should remain part of the demo, not be skipped
- visible LLM wiki growth is part of the core demo value
- private real relationship data must not be exposed in remote-safe artifact layers
