# Ops Spec

## Operational Requirement
v4 must include an ops layer, not just product/runtime/wiki logic.
The operating assumption is that the user controls the Mac mini Hitch environment from a MacBook.

## Ops Direction
Provide structure and docs for:
- how the system is run
- how status is checked
- how it is restarted
- how it is recovered
- how remote operator control works

## Atlas Guidance Role
Atlas may use accumulated background understanding from prior product conversations when:
- guiding implementation
- unblocking ambiguous choices
- reviewing Codex output
- detecting drift
- reviewing product tone and manner

The spec does not need to be 100 percent closed before implementation as long as Atlas actively provides product-intent guidance and drift control.
