# Relationship Space Spec

## Scope Simplification
- v4 should avoid overcomplicating multi-relationship orchestration at first
- immediate real target is one active romantic-partner relationship

## Core Unit
Preferred term:
- relationship space

A relationship space is the independent operating context for one relationship.

It owns:
- daily flow
- weekly flow
- memory accumulation
- summaries
- interpretation logic

Multiple relationship spaces may exist later, while the wiki layer remains connected underneath.

## Hitch-Per-Relationship Model
- each relationship has its own Hitch context operationally
- the broader LLM wiki may still be connected across contexts

## Creation Timing
- create the relationship space after a small amount of basic setup information is collected
- not immediately at first selection tap
- not only after the entire love-language onboarding is finished

## Minimum Creation Set
1. relationship type
2. what the user calls the other person
3. one-line current relationship state
4. preferred daily question delivery time
5. simple vs deep love-language test choice
