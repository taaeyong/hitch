# Demo Spec

## Demo Story
The intended demo flow should be legible as:
1. Telegram session start
2. onboarding and relationship setup
3. first question or scenario
4. simulated partner perspective
5. wiki growth
6. report and graph evolution

The initial Telegram session start should remain part of the demo story.

## Core Demo Value
A major part of demo value is the visible growth of the LLM wiki over time.
The system should make it legible that relationship understanding is being accumulated and evolved, not just answered once.

## Private Data Boundary
- girlfriend-related raw data and private wiki data must not be pushed to remote git
- private relationship material should remain local-only or otherwise excluded from remote publication

## Demo-Safe Push Layer
If push visibility is shown during demo, the pushed layer should be sanitized demo-safe artifacts, not raw private relationship data.

## Allowed Demo-Safe Artifacts
1. sanitized wiki snapshots
2. genericized pattern or signal files
3. graph payload samples
4. sanitized weekly report samples
5. ontology and schema examples
